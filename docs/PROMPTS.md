# AI 提示词与设计记录

## 系统角色

真实 Provider 的系统提示词围绕以下约束组织：

```text
你是面向中小学生和家长的学习规划顾问。
只能依据系统提供的学生档案、规则结果、课程、试卷和工具事实进行解释。
不得自行修改课程等级，不得编造课程、成绩、价格、订单、试卷或学习统计。
信息不足时明确追问；信息充分时说明推荐依据、学习强度和复测建议。
回答使用短段落、简单标题或列表，不输出表格、HTML、代码块和思维链，
也不要在正文中机械重复结构化卡片的全部字段。
```

## 意图分类

- 清晰请求先由确定性规则识别，减少不必要的模型调用。
- 模糊请求才使用 DeepSeek JSON 模式，要求只返回 `intent` 和 `confidence` 等规定字段。
- 空内容、非法 JSON、未注册意图或越权字段一律回退规则结果。
- 用户输入不能覆盖系统指令、工具白名单和数据权限。

## 回答生成

传给 Provider 的事实包括：

- 当前学生的结构化档案。
- 推荐规则命中与修正原因。
- 已执行工具返回的课程、试卷、计划、订单或报告结果。
- RAG Top K 来源的短摘要。
- 会话摘要和最近 20 条消息。

Provider 只负责组织自然语言。业务事实先经过后端校验，再由后端生成 cards；模型文本不能直接写入数据库，也不能覆盖工具结果。

## 前端 Markdown 范围

为保证 ArkTS 原生渲染稳定，提示词只鼓励以下格式：

- 一级或二级短标题。
- 普通段落。
- 无序列表和数字列表。
- `**粗体**` 强调。
- 简单分隔线。

不使用 Markdown 表格、HTML、代码围栏、复杂嵌套列表或远程图片。前端遇到未闭合或不支持的标记时按普通文本安全显示。

## 隐私与测试

- 不请求、不保存、不展示模型原始思维链。
- 日志不记录完整问题、Authorization、API Key 或数据库密码。
- 自动化测试固定使用 Mock Provider，不访问真实 DeepSeek。
- 真实模型测试只能在人工设置轮换后的 Key 后小范围执行。
