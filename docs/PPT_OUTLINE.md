# 答辩 PPT 素材

## 1. 项目背景

- 家长难以判断孩子适合基础、提升还是拔高课程。
- 课程、试卷、错题和学习计划通常彼此割裂。
- 通用聊天模型容易给出无法追踪、无法验证的推荐。

## 2. 项目方案

```text
学生档案
→ 可解释课程分层规则
→ AI 意图与工具编排
→ 课程/试卷推荐
→ 在线练习与错题
→ 学习计划与报告
```

面向小学五、六年级数学和英语构建首版闭环，避免一开始覆盖全部年级和学科。

## 3. HarmonyOS 前端亮点

- 单一 `Navigation + NavPathStack + PageMap`，四个主入口和无底部导航的业务子页面。
- 现代教育平台首页：Swiper、学习概览、AI 推荐、热门课程、最新课程和专项训练。
- 完整个人中心：课程、订单、收藏、错题、报告、客服、设置和家庭学生信息。
- AI 学习顾问：SSE、轻量 Markdown、消息级来源、四类结构化卡片、历史恢复和错误重试。
- 端侧语音识别、TTS、相册选图和 OCR。

## 4. 后端与数据

- FastAPI + Pydantic v2 + SQLAlchemy + MySQL + Alembic。
- JWT、PBKDF2-SHA256、家庭隔离、独立家长/学生账号和一次性绑定码。
- 12 门课程、24 份试卷、真实题目、练习记录、错题和学习报告聚合。

## 5. AI 架构亮点

- 13 类意图和 8 个显式工具，不是裸调用大模型。
- 规则决定课程等级，模型负责解释，推荐结果可追踪。
- 中文 n-gram TF-IDF RAG，来源可在前端展开查看。
- 会话摘要 + 最近消息控制上下文长度。
- cards 中的 ID、价格、成绩、状态全部来自数据库或工具。
- DeepSeek/Mock 统一 Provider，缺 Key 或异常时可控降级。

## 6. 可靠性与安全

- SSE 首段失败复用同一 `clientMessageId` 降级普通接口，避免重复业务操作。
- 后端异常时首页/个人中心保留 Mock，AI 页面保留问题并允许重试。
- 真实密钥不进入前端和 Git；测试不读取真实 Key。
- 不保存模型思维链，不让模型直接执行任意路由或覆盖业务事实。

## 7. 演示流程

1. 家长注册登录并创建学生档案。
2. 首页查看学习概览和智能推荐。
3. 使用文字或语音向 AI 咨询课程。
4. AI 追问缺失信息，调用课程和试卷工具。
5. 展示推荐理由、RAG 来源及课程/试卷卡片。
6. 进入课程详情或在线答题，提交后生成错题。
7. 将课程或试卷加入学习计划。
8. 在“我的”查看课程、订单、错题和学习报告。

## 8. 测试结果

- 后端 17 项自动化测试通过。
- HarmonyOS API 24 HAP 构建成功。
- MySQL 迁移与关键 API 冒烟完成。
- 真机语音、TTS、OCR、SSE 网络稳定性和签名安装列为最终设备验收项。

## 9. 项目边界

- 购买为模拟订单，不接真实支付。
- 云端任务同步不宣传为鸿蒙原生分布式流转。
- Mock Provider 和前端 Mock 会明确标记，不冒充真实模型或真实业务记录。
