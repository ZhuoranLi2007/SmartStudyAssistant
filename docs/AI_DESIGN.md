# AI与推荐设计

处理流程：用户输入 → 意图分类 → 字段提取 → 学生档案补全 → 参数缺失追问 → 工具注册与执行 → 规则推荐 → 模型组织解释 → 保存推荐与工具日志。

支持意图：课程推荐、试卷推荐、学情分析、课程查询、学习计划和普通教育咨询。

推荐规则：低于60分为基础型，60–84分为提升型，85分以上为拔高型；三个及以上薄弱点降一级；“巩固基础”限定基础型；竞赛/拓展只有成绩不低于85且薄弱点不超过一项时允许拔高。

`MockModelClient` 是可测试的演示实现，不是真实大模型。`OpenAICompatibleModelClient` 是供应商边界，答辩若要求真实大模型，应在不改变业务工具的前提下接入DeepSeek等兼容服务，并补充真实调用测试。
